# 弾正戦国転生記 開発ルール・設計決定事項

## ⚠️ 絶対に守るルール

### コード変更
- `config.py` の `LLM_PROVIDER` は **絶対に変更しない**。変えてよいのは `LMSTUDIO_MODEL` のみ
- ユーザーが貼った座標・数値データは **一切変更せずそのまま使う**
- セーブファイル・データファイル（.json/.csv/.txt）の削除・上書きは **必ず確認を取る**
- ユーザーが「どうしようか」「あとで」と言っている段階で実装しない。「OK」「そうして」が出てから動く

### 城の追加・変更時は必ず4ファイル同時更新
1. `data/scenarios/hizen_1560.json` — 城定義・石高・兵力・イベント
2. `game_state.py` の ADJACENCY — 隣接関係
3. `map_adjacency.html` の castles オブジェクト — 地図表示・座標
4. `map_state.json` — ※ゲーム再起動で自動生成されるため恒久変更は hizen_1560.json に行う

### map_state.json の注意
- ゲーム起動時に hizen_1560.json を一度読み込んでメモリ保持
- "m"キー・ターン終了時に write_map_state() が上書き
- **手動編集しても次回ゲーム起動で上書きされる**
- hizen_1560.json 変更後は必ずゲームを再起動してからマップを開く

## 石高の表示ルール（全箇所統一）

- **1万石以上**: `X.X万石`（例: 3.2万石）
- **1万石未満**: `XXX石`（整数・3桁カンマ）（例: 630石、1,043石）
- map上・popup・tooltip すべてに適用

### 石高換算
- 天保→慶長換算係数: `0.4387`（309,935 ÷ 706,470）
- koku（万石）= 天保石高 × 0.4387 ÷ 10000
- 人口 = koku × 7000（鬼頭宏研究・1560年肥前推計）

## 地図（map_adjacency.html）のルール

- 中継地マーカー（白丸）は **✏ 編集モード中のみ表示**。通常時は非表示
- `loadAllWaypoints()` ではマーカーを作らずポリライン経路のみ復元する
- applyMapState が map_state.json をポーリング（2.5秒）して全城を自動更新する
- 石高フォーマットは applyMapState 内の形式（万石/石）に統一
- ルートは `{land, sea}` の二重チャンネル。クリックで9パターンをサイクル
- `DEFAULT_COASTAL_SEA` = 陸路+沿岸海路の両持ちデフォルト（旧DEFAULT_SEA）

## NPC・LLMのルール

- NPC武将は **戦国時代の人物として話す**。現代語・現代口調は絶対NG
- プロンプトには正しい例（「殿」「～でございます」）と禁止例（「あなた」「呼び捨て」）を両方書く
- ローカルLLMはルールが多いと混乱するため短く具体的に書く
- LLM_PROVIDER は起動時選択UI で切り替える設計。config.py のデフォルト値は触らない

## ゲーム設計の確定事項

### 基本
- 1ターン = 1ヶ月
- 評定ループ（複数行動）→ AI行動 → 月進行 → 収入/イベント

### 転生タイプ（3種）
| タイプ | 内容 |
|--------|------|
| 大名転生 | 大名本人。義元は軍師なし（雪斎1555年死去） |
| 軍師転生 | 大名に仕える軍師。主君死亡後も後継に仕えて継続 |
| 小豪族転生 | 弱小領主配下。参照小説と同コンセプト |

### 産業
- **転生者（プレイヤー）のみ建設可能**。NPCは現代技術を知らない
- 流下式塩田は転生者秘密技術（入浜式より3倍効率）
- 軍師が知っているのは入浜式の現状データのみ

### 兵種戦力換算
- 足軽 1.0 / 騎馬 1.5 / 鉄砲 2.0

### 港格収入
- tier1=10貫/月 / tier2=30貫/月 / tier3=80貫/月

### 港格と航路種別は別概念（重要）
- **港格（port_tier）** = 経済的価値。南蛮船が来るか、外洋貿易か、沿岸漁業レベルか
- **航路種別** = 隣接領地間の移動手段。沿岸海路・外洋海路・陸路は独立して設定する
- 港格が高くても接続ルートは沿岸海路で構わない（矛盾しない）

### 港格の確定値（肥前シナリオ）
| tier | 代表城 | 備考 |
|------|--------|------|
| 3 南蛮 | 勝尾嶽城 / 横瀬浦（1562年開港後）/ 口之津（有馬介入時） | |
| 2 外洋 | 日野江城・小佐々城・七ツ釜浦・俵石城・江川城・青方城 | |
| 1 沿岸 | 大村館・針尾城・口之津（初期）など | |
| 0 なし | 佐志方城・牛ノ岳城・内陸城郭 | |

- **横瀬浦（yokose）**: sawamori（沢森）領。大村主導で1562年7月に南蛮港として開港。1563年8月に焼討ちでtier0
- **七ツ釜浦（nanatukamaura）**: kosaza（小佐々）領・tier2外洋（南蛮ではない）

### 地図ルートシステム（二重ルート）
- 各接続は `{land, sea}` の2チャンネルを持つ
- 陸上の沿岸港は原則「陸路＋沿岸海路」の両持ち（`DEFAULT_COASTAL_SEA`に登録）
- 離島など陸路なしの接続のみ `DEFAULT_SEA_ONLY` に登録（現在は空）
- 外洋航路（`ocean`）は五島・壱岐など海峡を跨ぐルート用

### 3階層従属システム
- 大名 → 従属大名（小大名）→ 国人 のチェーン
- `liege` フィールド1つで表現。`all_indirect_vassals()` / `liege_chain()` でBFS辿る
- 階層例: 有馬（大名）→ 大村（従属大名）→ 小佐々純勝（国人）

### 史実進行
- プレイヤー以外は史実通りに進む
- プレイヤーの行動で歴史が変わる（これがゲームの核心）
- 固定イベントは year/month トリガーで自動発火
- skip_if_neutralized で干渉済みならスキップ

### warlord_type
- daimyo: 独立大名（AI行動あり）
- vassal: 従属国人（AI行動あり・プレイ可）
- clan: 一門・家臣（AI行動なし）

## 史実年表の3年チャンク注入システム

### ファイル
- `data/history/nenpo_hizen.txt` — 全史実年表
- `llm/history_context.py` — `build_nenpo_context()` が担当

### フォーマット
```
年/月|内容|requires:条件1,条件2
```
- 月は省略可（月不明の出来事は年だけ書く→ソート末尾扱い）
- `requires:` は省略可

### 動作
- `build_nenpo_context(year, month, state)` を毎AIターンに呼ぶ
- 現在年を基準に **3年チャンク**（BASE_YEAR=1560, CHUNK_SIZE=3）を計算
- 前チャンク・現チャンク・次チャンクの3ブロックをAIプロンプトに注入
- `requires:` 条件をゲーム状態と照合し、不満足なら「史実乖離」として表示

### 条件型
| 条件 | 意味 |
|------|------|
| `warlord_alive:ID` | 指定武将が生存している |
| `warlord_dead:ID` | 指定武将が死亡している |
| `relation_hostile:ID1,ID2` | 両者が敵対関係 |
| `koku_gt:ID,値` | 指定勢力の石高が値を超えている |

### チャンク境界
1559-1562 / 1563-1565 / 1566-1568 / 1569-1571 / 1572-1574 / 1575-1578 / 1579+

### nenpo_hizen.txt を編集するとき
- 月が判明した場合は必ず `年/月|` 形式で書く
- requires条件を追加するとAIが史実乖離を検知できる
- 編集後はゲーム再起動不要（毎ターン動的に読み込む）
